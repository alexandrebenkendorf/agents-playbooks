# Sections

This file defines sections for React testing rules.

---

## 1. Creating New Tests (new)

**Impact:** HIGH  
**Description:** Proper test structure and patterns ensure maintainable, reliable tests that catch bugs early and document expected behavior.

## 2. Common Patterns (common)

**Impact:** MEDIUM  
**Description:** Reusable testing patterns that improve test quality, reduce duplication, and speed up test development.

## 3. TypeScript Testing (typescript)

**Impact:** MEDIUM-HIGH  
**Description:** Type-safe testing patterns that catch type errors early and improve test maintainability.

## 4. Legacy Refactoring (legacy)

**Impact:** HIGH  
**Description:** Strategies for migrating from legacy testing tools (Sinon, Enzyme) to modern approaches (Vitest, React Testing Library).
